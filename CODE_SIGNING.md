# Code Signing Policy

EdgePeek is intended to be distributed as an open-source Windows desktop application.

## Signing Goal

Release builds should be signed so users can verify that downloaded binaries were produced from this public source repository and were not modified after release.

The preferred signing path is SignPath Foundation open-source signing. SignPath signs artifacts built by trusted CI workflows and links the signed binaries back to the source repository and workflow run.

## Release Artifacts

Release builds produce:

- `EdgePeek-<version>-win-x64-portable.zip`
- `EdgePeekSetup-<version>-win-x64.exe`

The installer is the primary distribution artifact. The portable zip is provided for users who prefer extract-and-run distribution.

## Build Rules

- Release artifacts are built on GitHub-hosted Windows runners.
- Release artifacts are generated by `.github/workflows/release.yml`.
- Pull requests build and test the project, but signing is limited to release workflows.
- Signing should only be enabled for reviewed code from the main repository.
- SignPath credentials must be stored as GitHub secrets, not committed to the repository.

## GitHub Configuration

After SignPath approves the project, configure these repository variables:

- `SIGNPATH_ORGANIZATION_ID`
- `SIGNPATH_PROJECT_SLUG`
- `SIGNPATH_SIGNING_POLICY_SLUG`

Configure this repository secret:

- `SIGNPATH_API_TOKEN`

The release workflow works without these values and uploads unsigned artifacts. Once the values are configured, the SignPath signing step is enabled automatically for tag or manual release runs.

The workflow uses the default SignPath artifact configuration. If SignPath assigns a non-default artifact configuration slug during review, add it to `.github/workflows/release.yml`.

## Verification

Windows users can verify signed files with:

```powershell
Get-AuthenticodeSignature .\EdgePeekSetup-0.1.8-win-x64.exe
```

or:

```powershell
signtool verify /pa /v .\EdgePeekSetup-0.1.8-win-x64.exe
```

Unsigned local development builds are expected to show `UnknownError` or `NotSigned`.
