# Security Policy

## Supported Versions

Security fixes are provided for the latest release.

## Reporting a Vulnerability

Please report security issues privately by opening a GitHub security advisory for this repository, or by contacting the maintainer through the repository owner profile.

Do not disclose exploitable issues publicly until a fix is available.

## Release Integrity

Official release artifacts are built by GitHub Actions. When SignPath signing is enabled, signed artifacts are produced from the release workflow artifacts and can be verified with Windows Authenticode tools.
